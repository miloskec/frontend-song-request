# Skill: Changelog Discipline

## Goal
Make every meaningful repo change discoverable and explainable.

## Process
1. Add the change under `Unreleased`.
2. State what changed.
3. State why it changed if not obvious.
4. Mention if specs/contracts/gates were touched.
5. Keep entries concise and factual.

## Best practices
- Do not leave major work undocumented.
- Do not write vague entries like "updates".
